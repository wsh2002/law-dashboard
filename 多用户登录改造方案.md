# 多用户登录改造方案（Supabase 版）

## 当前问题分析

- `src/components/Login.tsx`：硬编码 `admin/admin123`，所有人共用同一个账号
- `src/App.tsx` 第 214 行：localStorage 固定使用 `lawDashboardState` 这一个 key，不同用户会互相覆盖彼此的状态
- `src/services/firebase.ts`：是 MockFirebase，不涉及真实认证

## 架构变更

```
【改造前】
用户A ──┐
        ├──> Login 硬编码(admin/admin123) ──> localStorage: lawDashboardState (共用)
用户B ──┘

【改造后】
用户A ──> Supabase Auth ──> localStorage: lawDashboardState_uid_A (独立)
用户B ──> Supabase Auth ──> localStorage: lawDashboardState_uid_B (独立)
```

## 第一步（手动操作）：创建 Supabase 项目

> 国内可直接访问，约 5 分钟完成

1. 访问 https://supabase.com，用 GitHub 或邮箱注册账号
2. 点击 **New Project**，填写项目名称和数据库密码（随意填，记住即可）
3. 等待项目初始化（约1分钟）
4. 进入项目后，点击左侧菜单 **Authentication > Users**，点击 **Add User** 添加每个使用者的邮箱和密码
5. 进入 **Project Settings > API**，复制以下两个值：
   - **Project URL**（形如 `https://xxx.supabase.co`）
   - **anon public key**（一长串字符）
6. 将这两个值写入项目根目录的 `.env` 文件（新增以下内容）：

```
VITE_SUPABASE_URL=https://你的项目ID.supabase.co
VITE_SUPABASE_ANON_KEY=你的anon_key
```

## 第三步（代码改造）：修改三个文件

### 文件 1：`src/services/supabase.ts`（新建文件，替代 firebase.ts）

```typescript
import { createClient } from '@supabase/supabase-js';

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL;
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY;

export const supabase = createClient(supabaseUrl, supabaseAnonKey);
```

---

### 文件 2：`src/components/Login.tsx`

主要改动：
- 将"用户名"输入框改为"邮箱"（`type="email"`）
- `handleLogin` 改为调用 Supabase 的登录接口
- 错误信息显示中文

核心逻辑变化：
```typescript
// 改前
if (username === 'admin' && password === 'admin123') {
    onLogin();
}

// 改后
const { error } = await supabase.auth.signInWithPassword({ email, password });
if (error) {
    const errorMap: Record<string, string> = {
        'Invalid login credentials': '邮箱或密码错误',
        'Email not confirmed': '邮箱尚未验证',
        'Too many requests': '尝试次数过多，请稍后再试',
    };
    setError(errorMap[error.message] || '登录失败：' + error.message);
}
// 登录成功后 onAuthStateChange 会自动触发，无需手动调用
```

### 文件 3：`src/App.tsx`

**改动 a：用 Supabase Auth 状态替代 isLoggedIn**

```typescript
// 改前（第 236 行）
const [isLoggedIn, setIsLoggedIn] = useState(false);

// 改后
import { Session } from '@supabase/supabase-js';
import { supabase } from './services/supabase';

const [session, setSession] = useState<Session | null>(null);
const [authLoading, setAuthLoading] = useState(true); // 避免页面闪烁

useEffect(() => {
  // 初始化时获取当前 session
  supabase.auth.getSession().then(({ data: { session } }) => {
    setSession(session);
    setAuthLoading(false);
  });

  // 监听登录/登出状态变化
  const { data: { subscription } } = supabase.auth.onAuthStateChange((_event, session) => {
    setSession(session);
  });

  return () => subscription.unsubscribe();
}, []);
```

**改动 b：localStorage key 改为用户专属**

```typescript
// 改前（第 214、227 行）
localStorage.getItem('lawDashboardState')
localStorage.setItem('lawDashboardState', ...)

// 改后（每个用户有独立存储空间，互不干扰）
const userId = session?.user?.id;
const stateKey = `lawDashboardState_${userId}`;
localStorage.getItem(stateKey)
localStorage.setItem(stateKey, ...)
```

**改动 c：条件渲染**

```typescript
// 改前
if (!isLoggedIn) return <Login onLogin={() => setIsLoggedIn(true)} />;

// 改后
if (authLoading) return <div>加载中...</div>; // 等待 Supabase 初始化
if (!session) return <Login />;
```

**改动 d：添加登出按钮**

在顶部导航栏添加：
```typescript
<button onClick={() => supabase.auth.signOut()}>退出登录</button>
```

## 关键优势

| 特性 | 改造前 | 改造后 |
|------|--------|--------|
| 账号数量 | 只有 admin 一个 | Supabase 后台无限添加 |
| 登录方式 | 硬编码校验 | 真实邮箱+密码认证 |
| 刷新后登录状态 | 丢失，需重新登录 | 自动保持（token 自动刷新） |
| 用户数据隔离 | 所有人共用同一份状态 | 每人独立存储，互不影响 |
| 用户管理 | 需要改代码 | Supabase 后台直接操作 |
| 国内访问 | — | 无需VPN，直接访问 |
| 服务器需求 | 无 | 无（Supabase 托管） |
| 费用 | 免费 | 免费（50,000用户上限） |

## 注意事项

- Supabase 免费项目若连续 **7 天无活跃**会自动暂停，重新访问项目后台点"恢复"即可（几分钟内恢复）

- `anon key` 是公开安全的，可以放在前端代码里，不是密钥
